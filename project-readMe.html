01Project Overview

What This Project Is

This project presents a federated learning approach for story point estimation in software development. Instead of pooling all project data into a single repository, participating teams train locally and share only model updates. The objective is to improve sprint planning quality while preserving organizational privacy boundaries.

Problem It Solves

In many companies, project histories contain sensitive internal information and cannot be directly shared. The core challenge is: how to train a useful estimation model when raw data must stay local. This system addresses that by enabling collaborative learning without exposing private project records.

Problem statement: As a software company, I do not want to share my project data.

03Methodology / Workflow

The system follows a federated training cycle where each participant contributes to a shared global model without exposing raw records.

Initialize global model
A base transformer model is selected and distributed to all participating clients.
Local client training
Each client trains the model on its own project dataset for a fixed number of local epochs.
Share model updates only
Clients return parameter updates (not raw project data) to the federated server.
Server-side aggregation
The server combines updates into a new global model using federated optimization rules.
Repeat across rounds
The process continues for multiple global epochs until performance stabilizes.
Evaluate and predict
The final model is tested on held-out data and used to estimate story points from new inputs.

08Limitations and Future Work

Current Limitations

Potential constraints include non-IID client data, communication overhead across many rounds, and variation in client compute resources. Model generalization may also vary across projects with very different text styles.

Future Improvements

Next steps may include advanced aggregation strategies, personalization layers per client, stronger privacy mechanisms, and broader benchmark comparisons across centralized vs federated settings.

09Conclusion

This project demonstrates that story point estimation can be approached in a privacy-preserving manner through federated learning. By combining decentralized training with transformer-based NLP, teams can collaborate on model quality while keeping internal project data local. The framework is practical for organizations that require both estimation intelligence and strict data confidentiality.
