EXPERIMENTAL SETUP
==================

Task
----
Story point estimation for software issues (TAWOS dataset).
Each Agile project is treated as a separate federated client, so no raw
issue data is ever shared across projects.

Dataset & Partitioning
-----------------------
- Source : TAWOS dataset (Jira issues with story point labels)
- Clients: 39 projects, one CSV file per project
- Split  : 80 / 20 train-test split applied independently per client
- Input  : issue title + description (concatenated with [SEP]), issue type,
           issue priority
- Target : story point value, z-score normalised per client

Model Architecture
------------------
A hybrid neural network (StoryPointRegressor) combining:
  1. Text encoder  — BERT-tiny (prajjwal1/bert-tiny, 4.4 M parameters)
                     Mean-pooled over valid tokens to produce a 128-dim vector.
  2. Categorical embeddings — learned 16-dim embeddings for issue type and
                     priority, concatenated with the text vector.
  3. Regression head — LayerNorm → Linear(160, 128) → ReLU → Dropout(0.2)
                     → Linear(128, 1)
  Loss : MSELoss on normalised targets
  Output : denormalised per-client story point prediction

Federated Learning Protocol
----------------------------
Algorithm : FedAvg with sample-count-weighted averaging of model parameters.
Rounds    : configurable (default 8, experiments run up to 20)
Clients/round: configurable fraction (default 25 %; full participation tested)
Local epochs : configurable (1, 3, or 10 per round tested)
Optimiser : AdamW, lr = 2e-5, weight_decay = 1e-4, batch_size = 16
Device    : CUDA (NVIDIA GPU, CUDA 12.4)
Seed      : 42 (fixed for reproducibility)

Evaluation Metrics
------------------
MAE, RMSE, MAPE, R²  (computed on raw, denormalised story point values)
Baseline: global mean predictor

Major Libraries
---------------
PyTorch 2.6 (cu124)
  Core deep learning framework. Used for model definition, training loop,
  gradient computation, and GPU acceleration via CUDA 12.4.

Hugging Face Transformers 4.57
  Provides the BERT-tiny pretrained model (AutoModel) and tokenizer
  (AutoTokenizer) with padding, truncation, and attention-mask handling.

Hugging Face Tokenizers 0.22
  Fast Rust-backed tokenizer used under the hood by Transformers for
  efficient batch encoding.

scikit-learn 1.8
  Used for per-client stratified train/test splitting (train_test_split).

pandas 3.0
  Data ingestion (CSV and Excel), column mapping, per-client groupby
  operations, and preprocessing.

NumPy 2.3
  Numerical operations throughout: normalisation statistics, metric
  computation, and prediction aggregation.

SciPy 1.17
  Available for additional statistical utilities (not yet heavily used
  directly but pulled in by scikit-learn).
